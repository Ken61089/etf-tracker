## 儀表板有什麼

### 🤝 共同動作（第一個分頁）
跨 ETF 的法人共識分析，分三塊：
1. **🔴 多檔同步買進 / 新增** — 同一天有 ≥2 檔 ETF 一起買進的個股（新進場標 🆕）
2. **🟢 多檔同步賣出 / 剔除** — 同一天有 ≥2 檔 ETF 一起賣出的個股（出清標 ✖）
3. **⚖️ 分歧** — 同一檔同一天「有人買、也有人賣」的個股

每筆都列出**幾家、是哪幾家、各買賣多少股**。家數越多排越前面。
（共識門檻目前為 2 家，可調整。）

### 各 ETF 分頁（每檔一頁）
1. **新增標的** — 相較上一份新買進的個股
2. **剔除持股** — 上一份有、現在完全沒持有的
3. **今日買入前五** — 持有股數增加最多的前 5
4. **今日賣出前五** — 持有股數減少最多的前 5

另可展開該檔的完整持股清單。

> 顏色依台股慣例：🔴 紅＝買進／新增，🟢 綠＝賣出／剔除。

## 比對怎麼運作（重要觀念）

MoneyDJ 網頁**每次只顯示「最新一天」**的持股。能做到日對日比較，是因為：

> **每次抓取，系統就把當天快照存進 GitHub repo 永久保留**（`data/<代號>/<資料日期>.json`），
> 時間久了累積成一份自己的歷史資料庫。比對時拿「最新這份」對比「上一個**不同資料日期**的快照」。
> 網站只負責給今天，歷史是我們自己留的。

由此衍生的特性：
- **以「資料日期」為基準，不是日曆日**。ETF 持股延遲公布，同一天多跑幾次不會重複計算；週末／沒更新的日子會覆寫同一份，不會產生假變化。
- **需要 ≥2 個不同資料日期才會有比對結果**。第一次跑只有一份，會標示「首次建立基準」，等下一次資料日期更新後就自動出現變化。
- **萬一某天漏抓**，下次會拿最新對比「上次存到的那份」，呈現跨那幾天的淨變化；每張卡片都會標出**實際比對區間**（例如「對比 06-05 → 06-09」），完全透明。
- **歷史不會遺失**，每份快照都進版控永久保存。

## 自動更新（線上版）

已設定 GitHub Actions（見 `.github/workflows/update.yml`）：

- **每天台灣時間 18:00**（cron UTC 10:00）自動抓取 → 比對 → 產生儀表板 → commit 回 repo → GitHub Pages 自動發布。
- 完全不需開電腦、不需手動。排程時間不保證分秒準時（尖峰可能延遲十幾分鐘，屬正常）。
- 也可在 GitHub 的 **Actions 分頁 →「每日更新 ETF 持股」→ Run workflow** 手動立即觸發。

## 手動使用（本機）

最簡單：在 Finder 對 **`更新並開啟.command`** 點兩下 → 自動抓最新、產生儀表板並打開。

> 第一次點若出現「無法打開，因為來自未識別的開發者」，到
> 系統設定 → 隱私權與安全性 → 點「仍要打開」，之後就不會再問。

或用終端機：
```bash
cd ~/etf-tracker
python3 etf_tracker.py          # 抓最新資料 + 產生儀表板
python3 etf_tracker.py --build  # 不重抓，只用既有快照重產儀表板
open docs/index.html            # 開啟儀表板
```

本機手動與線上自動兩者並存、互不影響（僅需 Python 3 + curl，無第三方套件）。

## 怎麼加 / 移除 ETF

編輯 `etf_tracker.py` 最上面的 `ETFS` 字典，加一行 `"代號": "名稱",`（名稱隨意，會自動更新）：
```python
ETFS = {
    "00981A": "主動統一台股增長",
    # ... 加在這裡，例如：
    "00XXXX": "",
}
```
存檔後執行一次 `python3 etf_tracker.py`，再 `git add -A && git commit && git push` 即可。
（限 MoneyDJ 上「全部持股」格式相同的 ETF。）

## 容量與費用

- **線上網站固定約 84 KB**（只放最新一天，不會長大）。
- 每份快照約 4.4 KB、一個交易日（7 檔）約 31 KB。
- GitHub 歷史（.git）每年約成長 13 MB —— **約 78 年才會接近 GitHub 建議的 1 GB**。
- 公開 repo 的 Actions 免費無限分鐘、Pages 流量 100 GB/月。
- 結論：容量、流量、費用**幾十年內都不是問題，全程免費**。
