# 主動型 ETF 每日持股追蹤系統

追蹤 MoneyDJ 上三檔主動型 ETF 的每日持股變化，產生網頁儀表板。

## 追蹤標的
- 00981A 主動統一台股增長
- 00403A 主動統一升級50
- 00982A 主動統一全球創新

## 每天怎麼用（最簡單）
在 Finder 裡對 **`更新並開啟.command`** 點兩下，就會自動抓最新資料、產生儀表板並打開。

> 第一次點如果出現「無法打開，因為來自未識別的開發者」，請到
> 系統設定 → 隱私權與安全性 → 點「仍要打開」，之後就不會再問。

## 或用終端機
```bash
cd ~/etf-tracker
python3 etf_tracker.py        # 抓最新資料 + 產生儀表板
python3 etf_tracker.py --build  # 不重抓，只用既有快照重產儀表板
open docs/index.html          # 開啟儀表板
```

## 儀表板顯示什麼
每檔 ETF 各一頁，分四個區塊：
1. **新增標的** — 相較上一份新買進的個股
2. **剔除持股** — 上一份有、現在完全沒持有的
3. **今日買入前五** — 持有股數增加最多的前 5
4. **今日賣出前五** — 持有股數減少最多的前 5

另可展開完整持股清單。

## 重要：比對基準是「資料日期」不是日曆日
ETF 持股是延遲公布的。本系統每抓一次，就用網頁上的「資料日期」存一份快照
（`data/<代號>/<資料日期>.json`），並和**前一個不同資料日期**的快照比較。
所以只有在 MoneyDJ 的資料日期更新後，才會出現新的變化；同一天多跑幾次不會重複計算。

第一次執行時因為沒有可比較的舊資料，會標示「首次建立基準」，四個區塊都會是空的，
**等資料日期更新後**就會自動出現變化。

## 檔案結構
```
etf-tracker/
├── etf_tracker.py     # 主程式：抓取、解析、比對
├── dashboard.py       # 產生網頁儀表板
├── 更新並開啟.command  # 點兩下執行
├── .github/workflows/ # GitHub Actions 每日自動更新設定
├── data/              # 每日快照（自動產生 / 版控保存）
└── docs/              # 儀表板 index.html + data.json（GitHub Pages 來源）
```

## 線上自動版（GitHub Pages + Actions）
本專案已設定 GitHub Actions，每天台灣時間 18:00 自動抓取、更新並發布到 GitHub Pages，
網址可公開分享給別人。設定檔見 `.github/workflows/update.yml`。
本機手動執行與線上自動執行兩者並存、互不影響。

## 之後想加自動排程（每天下午 6 點）
目前是手動執行。要改成每天自動更新，有兩條路：
- **本機 macOS**：用 launchd 設定每天 18:00 自動跑（需要那時電腦是開機/喚醒狀態）。
- **雲端 always-on**：部署到雲端（如 Zeabur）加 cron，24 小時自動更新、不依賴自己的電腦。
